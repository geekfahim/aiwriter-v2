8.)Field types.
This one can be bit tricky for beginners. Basically there are 2 types of inputs (textarea or input).

Textarea:
A textarea is a larger input field where users can enter multiple lines of text. It is represented by a rectangular box on a web page or application. Textareas are commonly used when you want to allow users to input longer or multiline text, such as paragraphs, comments, or descriptions. For example, if you want to ask GPT to generate a story based on a prompt, you might provide a textarea where the user can enter a few sentences or a paragraph describing the initial setup or characters.
Input type:
The input type is a more general way to receive user input. It can be used for various types of input fields, such as text, numbers, checkboxes, radio buttons, and more. The specific type of input field is defined using the "type" attribute. For example, if you want the user to input a single word or a short answer, you might use an input field with the type set to "text." If you want the user to input a numeric value, you might use an input field with the type set to "number."
The choice between textarea and input type depends on the nature of the input you want to collect from the user. If you want longer or multiline text, a textarea is more suitable. If you need a shorter or specific type of input, you can use an input field with the appropriate type.
Ultimately, the choice of textarea or input type depends on your specific prompt requirements and the type of input you want to receive from the user.

Plan Settings

To create a subscription plan in Stripe, you must first set up Stripe if you haven't already done so. Navigate to the integrations section to get started. In Prompt Engineer SaaS as Admin you will be able to create many plans with many different use cases and differences between them. It will also allow you to create plan package where you can combine Amount of prompts and access to AI tools in the same plan.

Configuring Plan Basics:
Plan name - Name your plan
Description – Describe your plan in short
Monthly Price – Configure monthly price of your plan. It will enable all features you will set up later on
Yearly price - Configure monthly price of your plan. It will enable all features you will set up later on

Configuring Plan Features:

AI writer and chat functions - Words limit:

Limit amount of words your client will have at disposal by entering amount as number. Please also do your research to identify how much tokens will be spend for given amount of words as this will consume OpenAI API tokens you will integrate in settings with API. For unlimited amount of words leave it empty. By configuring to 0 it will set 0 words amount available in plan and therefore block your client to use this feature.

Image generator - Image count:

Here you can configure how much images your client can generate in given plan. Setting it up to 0 will result in 0 images per plan while leaving it blank will enable user to generate unlimited amount of words.

Text to speech function - Text to speech minutes:

this function is for transcriber built in system, same configuration rules as up applies on this configuration

Setting up only Prompts in plan:

1.) Max prompt amount:
This option governs how much prompts your Client can discover or use with this plan. Example you can set max amount of prompts to 200. Once this is set and if your client opts in for this plan it will start consume tokens you have configured to 200. So in other words every prompt user discovers will consume 1 token out of total 200 you have set up.

2.) Prompt Type:

This functionality will determine if prompts will be Free or Paid. Turning this to paid will configure plan and system to charge for each prompt discovered or better to say to consume tokens you have previously configured.

3.) Max prompt access:

By configuring this to yearly it charge plan once a year and your Client will be able to discover amount of prompts you have previously configured in given time period. Leaving it as it is initially set will configure plan and system to charge plan once a month. Please note prompts user has already discovered will be accessible to him in collections tab.

Integrations
Open AI
  1. Go to (https://openai.com/api) and create or login to your Open AI account.
  2. On the top right click Personal and from the dropdown select View API Keys
  3. On the resulting page click create new secret. Copy the secret and save it in your script.
Stripe
  1. Sign up for a Stripe account: Go to the Stripe website (https://stripe.com/) and click on the "Sign Up" button in the top right corner of the page. Follow the instructions to create an account.
  2. Go to your Stripe Dashboard (https://dashboard.stripe.com/) and then click on the "Developers" button on the right-hand side of the page.
  3. Select API Keys from the left side menu. You'll see two keys: a publishable key and a secret key. Copy them to your stripe integration.
  4. To set up webhooks, select Webhooks from the left side menu. Click the "Add endpoint" button to add a new webhook endpoint. On the next screen, enter the following url https://yoururl/webhook/stripe (replace yoururl with your domain name). Select the All events to listen to and click Add Events. Click Add Endpoint. Copy the Signing secret to webhook secret field.

That's it! You now have your Stripe publishable key, secret key, and webhook secret, which you can use to integrate Stripe into your application. Make sure to keep your keys and secrets safe and secure to protect your payment information.

Google
  1. Go to the Google Cloud Console and create a new project. https://console.cloud.google.com/apis/dashboard then follow the screen below: As you can see there is a pop-up box. Just check the Terms of service then click Agree and Continue.
  2. Then click the CREATE PROJECT to continue. Then input your project name and organization (you can set it as No organization) then click CREATE.
    NOTE: If you have a project already you can skip the above steps. And just select your current project name created previously.
  3. After creating the project, click the credentials link on the left side menu. Then click + CREATE CREDENTIALS button. From the dropdown select OAuth client ID.
  4. In the next screen click CONFIGURE CONSENT SCREEN.
  5. In the next screen click, select User Type as External
  6. If your account is new you need to finish the OAuth consent screen and provide the important details they ask for.
  7. Once the consent is finished, go back to the Credentials page by clicking Credentials on the left menu. Then click + CREATE CREDENTIALS button & from the dropdown select OAuth client ID (Like step 3 above)
  8. In the next screen set the following:
    • Application type = Web application
    • Authorized Javascript origins = {Your url e.g. https://yoururl.com}
    • Redirect Uri = {https://yoururl.com/google/callback}
    and then click Create and copy your Client ID and Secret from the popup box

That's it! You now have your Google Client ID and Secret, which you can use to integrate Google into your application. Make sure to keep your keys and secrets safe and secure to protect your users' data.
