Plan configuration and acsess in system

Registration logic and trial perioid

Trial perioid rules

Prompt Eingineer SaaS registration process is designed to offer both trial perioid or directly force your client into paid one. Main controll for trial perioid can overide your entire plans you have created if you navigate to Admin=>Settings=> General settings

If we set Trial perioid to 0, it will disable acsess to AI tools and to prompts. This means your client will be able to sign up=>enter to specific plan for prompts and than it will invoke pop up to upgrade and pay for plan to acsess prompt discover function or use them within AI tools

When you set trial perioid 1+ it will enable trial tokens to be used wich can be assigned to specific plan from Plan creation tool. It wont immideatly invoke plan upgrade pop up and users will be able to see prompt titles and description. In order to acsess prompt conetnt they will need to click discover button and add it to collections



If you have a trial period enabled and set >0 then you should add token amount >0. Make sure from the admin panel>Plans you add trial tokens if you want to make any prompt within tier acsessible. If you set it like this trial tokens will be atomaticaly added to your client ballance and they will be deducted from total amount once they click discover (+) button. Its important to note, if client consumes all free trial tokens he will need to purchase plan to obtain amount you designate to plan.

Paid plan Tokens

Admin can view, add, edit, remove, tokens from specific plan by navigating to Admin>Settings>Billing>Plans.

When you are defining tokens for plan its important to know once this tokens are consumed and client has subscribed to 1 month plan he will need to buy more within same plan. Here we can configure amount of tokens in plan on mothly basis or yearly base.

Configuring additional token price for prompts

Main purpose of additional prompt tokens is to enable user to buy more if he consumed all initial tokens in his plan beafore renewal date. This option created so you can better monetize your prompts



This function enables you to determine pricing of each induidual token client will purchase. Its important to note you are configuring price per 1 token and that there is no limit in number of tokens client can type to buy. You set $1 per token and client on his side types in input field 100 tokens. This will invoke payment processing and he will be charged total amount of $100 for 100 tokens. Copy to clipboard relation with prompts in plan mannager

This option enables you to secure your intellectual property. For now we have set it up like this so you can manage acsses. Also its important to note by defult prompt system is set up to not expose prompt content except Title, Description and category. This means AI prompt content field in your prompt creation tool will remain always hidden.

Turning on copy to clipoard witin plan creation will enable your client once he discovered prompt and consumed token to copy AI prompt field and export prompt out of SaaS. This is created so you can safeguard your prompts and eventually allow them only to be used within in app AI tools
